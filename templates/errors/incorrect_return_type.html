It looks like your code was unable to compile. Lets take a look why...

It appears that the return statement(s) in your code are stopping it from compiling.

This error tends to be because you are either:

  • Missing a return statement completely.
  • Returning a variable that is of the wrong type.
  • Having return statements within a a block (e.g. an if statement) which is not guaranteed to be executed and no return statement at the end of the method.

Example of missing a return statement

Incorrect Code

public static int sum (int a, int b) {
	int sum = a + b; // The sum is being calculated correctly, but is not being returned.
}

Correct Code

public static int sum (int a, int b) {
	int sum = a + b; 
	return sum; // The sum is being returned as required by the method header, and so will compile now
}

Example of returning a variable that is of the wrong type

Incorrect Code

public static String arraySum (int[] array)
{
	int sum = 0;
	for (int i = 0 ; i < array.length ; i++)
	{
		sum += array[i];
	}
	return sum; // The method header says that we need to return a string, but sum is an integer
	
}

Correct Code

public static String arraySum (int[] array)
{
	int sum = 0;
	for (int i = 0 ; i < array.length ; i++)
	{
		sum += array[i];
	}
	return Integer.toString(sum); // We can convert sum to a string and return it, so the return type matches what is specified in the method header
	
}

Example of having return statements within a a block

Inorrect Code

public static int maxOfTwoNumbers (int a, int b)
{
	if (a >= b) {
		return a;
	}
	// If the above condition doesn't return true then the method won't return an int like it should
}

Correct Code

public static int maxOfTwoNumbers (int a, int b)
{
	if (a >= b) {
		return a;
	}
	return b; // Adding this line at the end ensures that the method will always return an int regardless of conditions returning true
}
