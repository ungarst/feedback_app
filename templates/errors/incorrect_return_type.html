It appears that the return statement(s) in your code are stopping it from compiling.

This error tends to be because you are either:

  • Missing a return statement completely.
  • Returning a variable that is of the wrong type.
  • Having return statements within a a block (e.g. an if statement) which is not guaranteed to be executed and not having a return statement at the end of the method.
Incorrect Code Correct Code Explanation

Example

 
public int sum (int a, int b) {
  int sum = a + b; 
}
 
public int sum (int a, int b) {
  int sum = a + b; 
  return sum; 
}

The sum is being calculated correctly, but is not being returned. Returning the sum will allow allow the code to compile, as the method requires an int to be returned

public String arraySum (int[] array)
{
  int sum = 0;
  for (int i = 0 ; i < array.length ; i++)
  {
    sum += array[i];
  }
  return Integer.toString(sum);
}

The method header says that we need to return a string, but sum is an integer. We can convert sum to a string and return it, so the return type matches what is specified in the method header

Example

 
public int maxOfTwoNumbers (int a, int b)
{
  if (a >= b) {
    return a;
  }
}
 
public int maxOfTwoNumbers (int a, int b)
{
  if (a >= b) {
    return a;
  }
  return b;
}

If the if condition doesn't return true then the method won't return an int like it should. Adding the return statement at the end ensures that the method will always return an int regardless of the if condition.
